Tutorial de utilização do Publii

Configuração do Servidor

Como o Publii é um aplicativo de desktop, ele não precisa estar conectado ao servidor do seu site quando você estiver fazendo modificações ou adicionando conteúdo; em vez disso, sempre que suas alterações estiverem prontas para serem publicadas, você só precisará clicar no botão Sync Your Website no canto inferior esquerdo do aplicativo Publii:

Você pode sincronizar seu site com a frequência que desejar; cada vez que você sincronizar, o Publii comparará seus arquivos locais com os do servidor e carregará apenas as alterações, então, após a primeira sincronização, tudo será mais rápido. Antes de poder sincronizar seu site, você precisará inserir os detalhes do seu servidor no aplicativo. Não se preocupe se você ainda não tiver um servidor; você ainda pode usar o Publii para criar seu site e adicionar as informações do seu servidor mais tarde.

Primeiros Passos

Se você estiver configurando um servidor no Publii pela primeira vez, notará que o botão Sync Your Website não está disponível; em vez disso, você verá um botão marcado Configure Server. Clicar nele o levará para a tela Configuração do Servidor. Como alternativa, você pode clicar na opção Server na barra lateral esquerda para abrir esta tela também.

Se você ainda não tem um servidor, não se preocupe. Você pode continuar a criar conteúdo e personalizar seu site localmente no Publii e sincronizá-lo mais tarde, quando seu servidor estiver pronto.

Para configurar o Publii para sincronizar com o seu servidor, é essencial saber o tipo de servidor que será usado e ter as informações de acesso. Neste guia, vamos focar na configuração com FTP (File Transfer Protocol), uma opção padrão para muitos provedores de hospedagem.

Depois que seu servidor estiver configurado, você pode começar a criar o conteúdo do seu site; navegue pelo menu à esquerda para obter informações sobre como criar postagens, tags e autores, inserir imagens e vídeos e muito mais.

Escolhendo um tipo de servidor

Ao clicar no botão Configure Server, a primeira coisa que você verá é uma seleção de  Tipos de Servidor para você escolher; Selecione  FTP como o tipo de servidor. O Publii também oferece suporte a outros tipos de servidor, mas este guia abordará a configuração com FTP, por ser a opção mais prática. Há também uma opção para  Implantação Manual, que exporta os arquivos do site para sua máquina local.

Após escolher o FTP como tipo de servidor, você será levado para a tela de Server Configuration, onde poderá inserir as informações de conexão do seu servidor e prosseguir com a configuração.

Opções de configuração do servidor

Cada tipo de servidor que você escolher incluirá duas opções básicas e universais:

  • URL do site - Normalmente, será a URL inicial do seu site se você estiver executando seu próprio servidor, por exemplo, www.mysite.com ; você também pode escolher qual protocolo usar, incluindo http:// ,  https:// , file:/// , dat:// , ipfs:// , dweb:// ou nenhum protocolo se estiver buscando uma implantação manual em um site de intranet. As soluções de armazenamento em nuvem geralmente usarão seu próprio domínio gerado; nossos guias detalhados para cada tipo de servidor cobrirão onde encontrá-lo.
  • Use Relative URLs - Esta opção converte caminhos de URL completos em caminhos relativos, por exemplo, https://www.mysite.com/path/para/path/

Além dessas opções básicas, cada protocolo também tem seus próprios requisitos específicos de segurança, configuração e sincronização. Dependendo do protocolo escolhido, você pode ter muito mais opções disponíveis. Você precisará inserir as opções de configuração do protocolo selecionado e clicar no botão Save para armazenar os detalhes; se necessário, você também pode clicar no botão Test Connection para garantir que o Publii consiga se conectar e sincronizar com seu servidor. Agora, vamos dar uma olhada em cada um dos protocolos disponíveis e suas respectivas configurações separadamente:

FTPs

FTP (File Transfer Protocol) é usado ao usar um servidor fornecido por um host; se você comprou um plano de hospedagem de um host como GoDaddy ou Bluehost, então o Publii usará FTP para mover e gerenciar todos os arquivos do seu site para o seu servidor. Existem três tipos de FTP suportados pelo Publii; FTP padrão ou os mais seguros SFTP e FTPS. A maioria dos hosts suportará uma das opções mais seguras, mas em caso de dúvida, é melhor verificar com a equipe de atendimento ao cliente do seu host para garantir que você tenha a conexão mais segura possível.

O protocolo FTP usa uma transmissão não criptografada, o que significa que quaisquer dados enviados por ele, incluindo seu nome de usuário e senha, podem ser lidos por qualquer um que possa interceptar sua transmissão. Recomendamos fortemente usar os protocolos FTPS ou SFTP, se possível.

As opções disponíveis ao selecionar FTP como seu protocolo são:

  • Port - Define a porta pela qual o Publii deve se conectar; geralmente, ela é definida como o padrão '21' (para FTP) ou '22' (para SFTP), mas seu provedor de hospedagem pode informá-lo sobre as configurações necessárias.
  • Server - Coloque aqui a URL para acesso ao seu servidor FTP; este será um endereço web que começa com o prefixo ftp:// em vez de http:// .
  • Username - O nome de usuário para seu acesso FTP.
  • Password - A senha para seu acesso FTP. Ambas são necessárias para que Publii possa acessar o servidor. O texto inserido aqui será ocultado por motivos de segurança; para exibir o texto digitado, clique no texto "Mostrar senha" neste campo.
  • Your Key (SFTP somente com chave) - Defina o caminho para sua chave SFTP aqui; clicar no campo abrirá o explorador de arquivos; basta navegar e selecionar sua chave para continuar.
  • Key Passphrase (SFTP somente com chave) - Digite a frase-senha da sua chave SFTP aqui, se você tiver uma.
  • Always Ask For Password - Quando esta opção estiver ativada, você será solicitado a digitar a senha do FTP toda vez que sincronizar seu site, para maior segurança.
  • Remote Path - O caminho remoto é a pasta que seu FTP abre quando se conecta pela primeira vez; normalmente seria a pasta principal /public_html/ , mas pode ser que você tenha acesso limitado a uma subpasta específica, como public_html/mySite/ .

Manual Upload

Embora o Publii seja projetado para gerenciar atualizações e sincronizações de sites automaticamente, pode haver alguns usuários que não têm acesso direto ao seu servidor, preferem carregar seus sites manualmente ou apenas desejam gerar seus sites em um arquivo zip para implantação posterior. Nesses casos, os usuários podem aproveitar a opção de upload manual, que gera o site em um formato de sua escolha.

As seguintes opções estarão disponíveis se o Upload Manual estiver definido como seu protocolo:

  • Output Type - Define o formato de arquivo que o Publii deve exibir em seu site quando uma operação de sincronização for executada. Escolha entre um arquivo ZIP ou TAR compactado ou a pasta do site descompactada. Os arquivos de saída são armazenados no mesmo local que o local dos arquivos do seu site Publii; assim que a sincronização for concluída, você terá a opção de abrir esse local automaticamente.
  • Output Directory - Define onde o arquivo de saída deve ser colocado quando gerado. Clicar no campo vazio abrirá o explorador de arquivos para você escolher um local.
